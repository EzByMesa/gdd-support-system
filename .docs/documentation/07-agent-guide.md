# 07. Руководство агента

Это руководство предназначено для пользователей с ролью **AGENT** — сотрудников службы поддержки.

---

## Вход в систему

Войдите с учётными данными, выданными администратором (или через 1С, если настроена интеграция). После входа вы попадаете на страницу со списком тикетов.

---

## Список тикетов

Агент видит **все тикеты** в системе, независимо от того, кем они созданы и на кого назначены.

### Фильтры по статусу

Вкладки над списком:

| Вкладка | Что показывает |
|---------|---------------|
| Все | Все тикеты |
| Открытые | Новые, ещё без агента |
| В работе | Агент работает |
| Ожидают ответа | Ждёт ответа от пользователя |
| Решённые | Проблема решена |
| Закрытые | Финальный статус |

### Дополнительные фильтры (параметры API)

При работе с API можно фильтровать по `assignee=me` (только ваши тикеты) или `assignee=none` (без назначения). В интерфейсе эти фильтры доступны через URL-параметры.

### Readonly-тикеты

Тикеты, не назначенные на вас, помечены бейджем **"Только чтение"**. Вы можете просматривать их описание и историю сообщений, но не можете отправлять сообщения в чат.

---

## Взятие тикета в работу

Чтобы начать работу с тикетом:

1. Откройте тикет из списка
2. В правой панели нажмите кнопку **"Взять в работу"** (или "Назначить на себя")
3. Статус тикета изменится на `IN_PROGRESS`, вы станете assignee

После назначения:
- Тикет становится доступным для чата с вашей стороны
- Пользователь получает уведомление (если включены уведомления)
- Псевдоним генерируется при отправке первого сообщения

### Изменение статуса вручную

В правой панели тикета есть выпадающий список **"Статус"**:

| Статус | Когда использовать |
|--------|-------------------|
| `IN_PROGRESS` | Вы работаете над проблемой |
| `WAITING_FOR_USER` | Задали вопрос, ждёте ответа от клиента |
| `RESOLVED` | Проблема решена |
| `CLOSED` | Обращение закрыто |

### Изменение приоритета

В правой панели тикета выберите приоритет: `LOW`, `MEDIUM`, `HIGH`, `CRITICAL`.

---

## Чат с пользователем

### Как видит чат агент

В интерфейсе агента отображается:
- Реальное имя пользователя
- Реальное имя агента (ваше) + псевдоним в скобках

Пример:
```
Иван Иванов (Ласковый Котёнок): Добрый день! Когда ожидать решения?
Пётр Петров [вы, псевдоним: Ласковый Котёнок]: Работаем над этим!
```

> Пользователь видит только псевдоним — "Ласковый Котёнок". Ваше реальное имя остаётся скрытым.

### Отправка сообщений

1. Введите текст в поле ввода
2. Нажмите **"Отправить"** или **Enter**

### Прикрепление файлов к сообщению

1. Нажмите кнопку скрепки (или иконку вложения)
2. Выберите файл (максимум 50 МБ)
3. После загрузки файл прикрепится к следующему сообщению
4. Отправьте сообщение

### Индикатор набора

Если пользователь печатает ответ, вы увидите индикатор "Иван Иванов печатает...".

## Делегирование тикета

Делегирование позволяет передать тикет другому агенту, например при уходе в отпуск или необходимости привлечения более компетентного специалиста.

### Создание запроса делегирования

1. Откройте тикет, который назначен на вас
2. В правой панели нажмите кнопку **"Делегировать"**
3. В открывшейся модалке:
   - Выберите агента из списка (или найдите по имени)
   - Напишите комментарий (опционально): почему передаёте, что уже сделано
4. Нажмите **"Отправить запрос"**

Выбранный агент получит уведомление в реальном времени.

### Ограничения

- Нельзя делегировать тикет, который не назначен на вас
- Нельзя делегировать самому себе
- Пока есть активный `PENDING` запрос — нельзя создать ещё один

### Исходящие запросы

На странице **"Делегирование"** (`/delegations`) вкладка **"Исходящие"** показывает ваши запросы и их статус:
- `PENDING` — ожидает ответа
- `ACCEPTED` — принято, тикет переназначен
- `REJECTED` — отклонено

### Входящие запросы

Вкладка **"Входящие"** показывает запросы, направленные вам. Счётчик активных запросов отображается рядом с пунктом меню.

**Для каждого запроса:**
- Информация о тикете (номер, заголовок, статус)
- Имя агента, который передаёт
- Комментарий
- Кнопки **"Принять"** и **"Отклонить"**

**При принятии:**
- Тикет переназначается на вас
- Генерируется ваш псевдоним для этого тикета
- Пользователь видит: "Теперь вашим обращением занимается [ваш псевдоним]"

**При отклонении:**
- Тикет остаётся у прежнего агента
- Прежний агент получает уведомление об отклонении

## Тематические группы

Страница **"Группы"** (`/topic-groups`) доступна агентам в режиме просмотра.

### Что показывает

- Список тематических групп с количеством тикетов
- Название группы (авто-генерированное или переименованное администратором)
- При нажатии на группу — список тикетов в ней

### Как использовать

Тематические группы помогают найти похожие обращения. Если вы видите, что много тикетов в одной группе — возможно, есть системная проблема, требующая отдельного решения.

> Переименование, объединение и удаление групп доступно только администраторам.

---

## Уведомления агента

Агент получает следующие типы уведомлений:

| Тип | Когда |
|-----|-------|
| Новое сообщение | Пользователь ответил в тикете, назначенном на вас |
| Запрос делегирования | Кто-то предлагает вам тикет |
| Делегирование принято | Агент принял ваш запрос |
| Делегирование отклонено | Агент отклонил ваш запрос |

Уведомления отображаются в колокольчике и приходят в реальном времени через WebSocket.

## Советы по работе

**Рекомендуемый рабочий процесс:**
1. Просматривайте вкладку "Открытые" — это новые тикеты без агента
2. Берите тикет в работу (`assigneeId = вы`, статус `IN_PROGRESS`)
3. Общайтесь с пользователем в чате
4. Если нужен ответ от пользователя — переводите в `WAITING_FOR_USER`
5. После решения — `RESOLVED`
6. Финальное закрытие — `CLOSED`

**При уходе в отпуск:**
- Используйте делегирование для передачи активных тикетов
- В комментарии укажите контекст: что уже сделано, какой следующий шаг

**Поиск похожих обращений:**
- Проверьте тематическую группу тикета — там могут быть аналогичные случаи с готовым решением
