# 08. Руководство администратора

Это руководство предназначено для пользователей с ролью **ADMIN**.

---

## Доступ к панели администратора

Панель администратора доступна по адресу `/#/admin` (автоматически перенаправляет на `/#/admin/dashboard`).

В боковом меню появляется раздел **"Администрирование"** с подпунктами:
- Дашборд
- Пользователи
- Тикеты
- Тематические группы
- Авторизация
- Настройки

Доступ к `/admin` строго ограничен ролью `ADMIN`. При попытке зайти с ролью `USER` или `AGENT` — перенаправление на главную.

---

## Дашборд (`/admin/dashboard`)

Дашборд отображает:

### Статистика

| Показатель | Описание |
|-----------|----------|
| Всего тикетов | Общее количество в системе |
| Открытых | Статус `OPEN` |
| В работе | Статус `IN_PROGRESS` |
| Решённых | Статус `RESOLVED` |
| Пользователей | Все учётные записи |
| Агентов/Админов | Пользователи с ролью `AGENT` или `ADMIN` |
| Тем. групп | Количество тематических групп |

### Последние тикеты

Таблица 10 последних тикетов с номером, заголовком, статусом и датой. Клик по тикету открывает его детальную страницу.

## Управление пользователями (`/admin/users`)

### Просмотр списка

Таблица пользователей с фильтрами:
- **Поиск** — по логину, отображаемому имени, email
- **Роль** — `USER`, `AGENT`, `ADMIN`
- **Статус** — активные/неактивные

Пагинация: 20 пользователей на странице.

### Создание пользователя

Нажмите **"Создать пользователя"**. Заполните форму:

| Поле | Обязательное | Описание |
|------|-------------|----------|
| Логин | Да | Латиница, цифры, `_`, `.`, `-`, мин. 3 символа |
| Пароль | Да | Мин. 8 символов |
| Отображаемое имя | Да | Имя, которое видят другие пользователи |
| Email | Нет | Адрес электронной почты |
| Роль | Нет | `USER` (по умолчанию), `AGENT`, `ADMIN` |

Пароль хранится в виде bcrypt-хэша (cost factor 12).

### Редактирование пользователя

Нажмите на пользователя или кнопку редактирования. Можно изменить:
- Отображаемое имя
- Email

Логин и пароль через этот интерфейс не меняются (пароль пользователь меняет самостоятельно).

### Смена роли

В меню пользователя выберите **"Изменить роль"**. Доступные значения: `USER`, `AGENT`, `ADMIN`.

> При понижении роли с `ADMIN` до `USER`/`AGENT` пользователь мгновенно теряет доступ к /admin при следующем запросе.

### Деактивация/активация пользователя

Кнопка **"Деактивировать"** устанавливает `isActive: false`. Деактивированный пользователь:
- Не может войти в систему
- Получает `403 ACCOUNT_DISABLED` при попытке входа
- Остаётся в базе данных (soft delete)

Кнопка **"Активировать"** восстанавливает доступ.

### Удаление пользователя

Кнопка **"Удалить"** деактивирует учётную запись (soft delete, `isActive: false`). Физическое удаление из БД не производится.

> **Важно:** Корневого администратора (`isRootAdmin: true`) нельзя удалить. Это защита от потери доступа к системе.

## Управление тикетами (`/admin/tickets`)

Администратор видит все тикеты системы. В отличие от обычного списка (`/`), здесь доступны расширенные фильтры:
- По статусу
- По приоритету
- По агенту
- Поиск по тексту

Администратор может:
- Открыть любой тикет
- Изменить статус, приоритет, назначение
- Отправлять сообщения в чат любого тикета
- Инициировать делегирование для любого тикета

## Настройки авторизации (`/admin/auth`)

### Провайдеры авторизации

Страница управления провайдерами входа в систему.

#### Локальный провайдер (LOCAL)

Всегда активен. Стандартный вход по логину и паролю. Не отображается как отдельный провайдер в таблице, но всегда присутствует в списке на странице входа.

#### Провайдер 1С (ONE_C)

Для настройки интеграции с 1С:

1. Нажмите **"Добавить провайдер"**
2. Выберите тип **"ONE_C"**
3. Введите название (будет показано на кнопке входа, например "Вход через 1С")
4. Укажите конфигурацию:

| Параметр | Описание | Пример |
|----------|----------|--------|
| `baseUrl` | Базовый URL сервера 1С | `http://1c-server:8080` |
| `authEndpoint` | Путь проверки учётных данных | `/auth/validate` |
| `timeout` | Таймаут запроса в мс | `5000` |

5. Нажмите **"Сохранить"**

#### Тестирование провайдера

После сохранения нажмите **"Проверить"** — система выполнит HEAD-запрос к `baseUrl + authEndpoint`. Результат: "Сервер 1С доступен" или "Недоступен: [ошибка]".

#### Включение/отключение провайдера

Переключатель **"Активен"** позволяет временно отключить провайдер без его удаления.

#### Удаление провайдера

Кнопка **"Удалить"** полностью удаляет провайдер. Пользователи, созданные через этот провайдер, сохраняются, но не смогут войти через него.

### Включение/отключение регистрации

Настройка `registration.enabled` управляет возможностью самостоятельной регистрации:
- `true` — кнопка "Зарегистрироваться" видна на странице входа
- `false` — самостоятельная регистрация недоступна (только создание через /admin/users)

Изменяется через **Системные настройки** → ключ `registration.enabled`.

## Системные настройки (`/admin/settings`)

Управление ключ-значение настройками системы.

### Доступные настройки

| Ключ | Тип | По умолчанию | Описание |
|------|-----|-------------|----------|
| `app.name` | string | `GDD Support System` | Название системы в интерфейсе |
| `registration.enabled` | boolean | `true` | Разрешить самостоятельную регистрацию |
| `storage.maxFileSize` | number | `52428800` | Максимальный размер файла (байты, 50 МБ) |
| `tickets.autoCloseAfterDays` | number | `7` | Автозакрытие решённых тикетов (дни) |
| `grouping.similarityThreshold` | number | `0.75` | Порог cosine similarity для ML-группировки |
| `storage.path` | string | — | Путь к папке вложений (задаётся при Setup) |
| `storage.encryptionKey` | string | — | Мастер-ключ AES-256 (задаётся при Setup) |
| `push.vapidPublicKey` | string | — | Публичный VAPID-ключ (генерируется автоматически) |
| `push.vapidPrivateKey` | string | — | Приватный VAPID-ключ (генерируется автоматически) |

> **Внимание:** Никогда не изменяйте `storage.encryptionKey` после загрузки файлов — все существующие вложения станут недоступными для расшифровки.

### Изменение настройки

1. Найдите нужный ключ в таблице
2. Нажмите кнопку редактирования
3. Введите новое значение
4. Сохраните

## Тематические группы (`/admin/topic-groups`)

Страница управления тематическими группами, создаваемыми ML-алгоритмом.

### Список групп

Таблица групп с:
- Названием (авто-генерированное или переименованное)
- Количеством тикетов
- Флагом `autoGenerated`

### Переименование группы

1. Нажмите кнопку редактирования рядом с группой
2. Введите понятное название (например, "Проблемы с принтерами" вместо "Принтер работает подключение")
3. Сохраните

### Объединение групп

Если ML создал несколько групп об одной теме:

1. Откройте целевую группу (куда переносить тикеты)
2. Нажмите **"Объединить с..."**
3. Выберите группу-источник
4. Подтвердите операцию

Все тикеты из группы-источника переносятся в целевую, группа-источник удаляется.

### Удаление группы

Тикеты группы теряют привязку (`topicGroupId = null`). При следующем прохождении переклассификации им будут назначены группы заново.

### Переклассификация всех тикетов

Кнопка **"Переклассифицировать все"**:

1. Удаляются все авто-генерированные группы (`autoGenerated: true`)
2. Все тикеты теряют привязку к группам
3. Тикеты обрабатываются последовательно через ML-модель
4. Создаются новые группы

Операция выполняется **в фоне**, не блокирует интерфейс. Прогресс отображается в логах сервера. При большом количестве тикетов (сотни/тысячи) операция может занять несколько минут.

> Вручную созданные и переименованные группы (у которых `autoGenerated: false`) **не удаляются** при переклассификации.

## Swagger — документация API (`/api/docs`)

Интерактивная документация API доступна по адресу:

```
http://localhost:3000/api/docs
```

### Требования доступа

- Войдите в систему как администратор
- В Swagger UI нажмите **"Authorize"**
- Введите: `Bearer <ваш_access_token>`
- Нажмите **"Authorize"**

Access token можно получить через `POST /api/auth/login` или скопировать из localStorage браузера (ключ зависит от реализации AuthService).

### Возможности Swagger UI

- Просмотр всех эндпоинтов с описанием параметров и ответов
- Тестирование эндпоинтов прямо из браузера
- Экспорт спецификации: `GET /api/docs.json`

## Безопасность

### Защита корневого администратора

Первый администратор (`isRootAdmin: true`) создаётся только через Setup Wizard. Его нельзя удалить через интерфейс и API. Это гарантирует наличие хотя бы одного администратора в системе.

### JWT токены

- Access token: 15 минут
- Refresh token: 30 дней, httpOnly cookie
- В production задайте сложный `JWT_SECRET` в `.env`

### Рекомендации для production

1. Задайте сложный `JWT_SECRET` (мин. 32 символа, случайный)
2. Используйте PostgreSQL вместо SQLite
3. Настройте HTTPS (через reverse proxy — Nginx или Traefik)
4. Регулярно делайте бэкап:
   - Базы данных
   - Папки вложений (`storage.path`)
   - Значения `storage.encryptionKey` из SystemSettings (без него вложения нельзя восстановить)
5. Установите `NODE_ENV=production` в `.env`

## Типичные административные задачи

### Добавить нового агента

1. `/admin/users` → **"Создать пользователя"**
2. Логин, пароль, имя, роль `AGENT`
3. Передайте логин и пароль агенту

### Повысить пользователя до агента

1. `/admin/users` → найти пользователя
2. **"Изменить роль"** → `AGENT`

### Отключить самостоятельную регистрацию

1. `/admin/settings`
2. Найти `registration.enabled`
3. Установить `false`

### Включить интеграцию с 1С

1. `/admin/auth` → **"Добавить провайдер"**
2. Тип: `ONE_C`, заполнить `baseUrl` и параметры
3. Протестировать подключение
4. Сохранить

### Тонкая настройка ML-группировки

1. `/admin/settings`
2. Найти `grouping.similarityThreshold`
3. Изменить значение (0.75 по умолчанию, диапазон 0.0–1.0)
4. При необходимости запустить переклассификацию в `/admin/topic-groups`
