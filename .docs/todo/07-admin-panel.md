# Фаза 8 — Админ-панель (отдельный крупный роут)

## Принцип

Админка — это **отдельный крупный роут** `/admin/...` с собственным layout, sidebar, и интерфейсом. Тот же SPA, тот же URL, но полностью отдельная визуальная оболочка.

Доступ: только `role === 'ADMIN'`. Роутер проверяет роль и редиректит на `/` если не админ.

**Полностью адаптивная** — работает на десктопе и мобильных.

## Разделы

### 1. Дашборд (`/admin/dashboard`)
- Карточки-счётчики: всего тикетов, открытых, в работе, решённых
- Мини-графики: новые тикеты за неделю/месяц
- Средне время ответа / время до решения
- Нагрузка по агентам (сколько тикетов у каждого)
- Топ-5 тематических групп
- Последние 10 тикетов

### 2. Управление пользователями (`/admin/users`)
```
GET    /api/admin/users              — список с пагинацией, фильтрами, поиском
POST   /api/admin/users              — создать пользователя
PUT    /api/admin/users/:id          — обновить
PUT    /api/admin/users/:id/role     — сменить роль (USER ↔ AGENT ↔ ADMIN)
PUT    /api/admin/users/:id/active   — активировать/деактивировать
DELETE /api/admin/users/:id          — удалить (soft delete)
```

Интерфейс:
- Таблица с пагинацией (на мобильных — карточки)
- Фильтры: роль, статус, провайдер авторизации
- Поиск по имени/логину/email
- Кнопка "Создать пользователя" → модальное окно
- Для каждого пользователя: смена роли (dropdown), активация/деактивация
- **Админ может назначать других Администраторов** (в том числе из агентов и пользователей)

### 3. Управление тикетами (`/admin/tickets`)
- Все тикеты с расширенными фильтрами (статус, приоритет, агент, дата, группа)
- Массовые действия: назначить агента, закрыть, сменить приоритет
- Экспорт в CSV
- Детальный просмотр с полной историей чата (без анонимизации — админ видит реальные имена)

### 4. Тематические группы (`/admin/topic-groups`)
- Список групп с количеством тикетов
- Создание/редактирование/удаление групп
- Объединение двух групп
- Кнопка "Переклассифицировать все" (фоновая задача)

### 5. Настройки авторизации (`/admin/auth`)
- Переключатель: регистрация вкл/выкл
- Список провайдеров авторизации (таблица)
- Добавление провайдера (LOCAL, ONE_C)
- Редактирование конфигурации 1С (URL, endpoint, timeout)
- Кнопка "Тестировать подключение"
- Активация/деактивация провайдеров

### 6. Системные настройки (`/admin/settings`)
```
GET /api/admin/settings              — все настройки
PUT /api/admin/settings/:key         — обновить настройку
```

Настройки:
- `registration.enabled` — регистрация
- `storage.maxFileSize` — макс. размер файла (МБ)
- `storage.allowedTypes` — допустимые MIME-типы
- `tickets.autoCloseAfterDays` — автозакрытие resolved тикетов (дни)
- `app.name` — название системы
- `grouping.similarityThreshold` — порог группировки (0.0-1.0)

Форма с полями, сохранение поштучно или "Сохранить все".

## Роутинг админки

```
/admin                    → redirect /admin/dashboard
/admin/dashboard          → AdminDashboardPage
/admin/users              → AdminUsersPage
/admin/users/:id          → AdminUserDetailPage
/admin/tickets            → AdminTicketsPage
/admin/tickets/:id        → AdminTicketDetailPage
/admin/topic-groups       → AdminTopicGroupsPage
/admin/auth               → AdminAuthPage
/admin/settings           → AdminSettingsPage
```

## Мобильная адаптация

- Sidebar → slide-in drawer (по бургер-кнопке)
- Таблицы → карточки на узких экранах
- Формы → полноэкранные на мобильных
- Модалки → bottom sheet на мобильных
- Touch-friendly элементы (мин. 44px target)
