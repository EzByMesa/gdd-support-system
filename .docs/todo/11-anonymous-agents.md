# Система анонимных агентов ("Милые животные")

## Концепция

Пользователь НИКОГДА не видит реальное имя агента. Вместо этого каждому агенту на каждый тикет назначается уникальный псевдоним — сочетание "Прилагательное + Животное".

## Алгоритм назначения

```javascript
// services/agentAlias.js

async function getOrCreateAlias(agentId, ticketId) {
  // 1. Проверяем — есть ли уже псевдоним для этого агента в этом тикете
  let alias = await AgentAlias.findOne({
    where: { agentId, ticketId }
  });

  if (alias) return alias.alias;

  // 2. Получаем все уже использованные псевдонимы в этом тикете
  //    (чтобы не повторяться, если тикет был делегирован)
  const usedAliases = await AgentAlias.findAll({
    where: { ticketId },
    attributes: ['alias']
  });
  const usedSet = new Set(usedAliases.map(a => a.alias));

  // 3. Генерируем уникальный псевдоним
  let newAlias;
  do {
    const adj = ADJECTIVES[Math.floor(Math.random() * ADJECTIVES.length)];
    const animal = ANIMALS[Math.floor(Math.random() * ANIMALS.length)];
    newAlias = `${adj} ${animal}`;
  } while (usedSet.has(newAlias));

  // 4. Сохраняем
  await AgentAlias.create({ agentId, ticketId, alias: newAlias });

  return newAlias;
}
```

## Правила анонимизации

### Что видит ПОЛЬЗОВАТЕЛЬ:
- В чате: "**Ласковый Котёнок**: Здравствуйте! Чем могу помочь?"
- В деталях тикета: "Ваш агент: **Ласковый Котёнок**"
- При делегировании: "Теперь вашим обращением занимается **Смелый Пингвин**"
- В уведомлениях: "**Ласковый Котёнок** ответил на ваше обращение #42"

### Что видит АГЕНТ / ADMIN:
- Реальное имя: "**Иванов Иван**: Здравствуйте!"
- + мелким шрифтом псевдоним: _(для пользователя: Ласковый Котёнок)_

## Сценарий: один тикет, несколько агентов

```
1. Агент "Иванов" берёт тикет → псевдоним "Ласковый Котёнок"
2. Иванов делегирует тикет "Петрову"
3. Петров подтверждает → новый псевдоним "Смелый Пингвин"
4. Пользователь видит: "Теперь вашим обращением занимается Смелый Пингвин"
5. Вся история чата сохраняется:
   - Старые сообщения от "Ласкового Котёнка" остаются
   - Новые от "Смелого Пингвина"
```

## Хранение в БД

Таблица `agent_aliases`:
- `agentId` + `ticketId` = unique (один агент = один псевдоним на тикет)
- При делегировании — новая запись для нового агента
- Старый псевдоним остаётся в истории (не удаляется)

## Безопасность

- Псевдоним НЕ содержит никакой информации о реальном пользователе
- Рандомный выбор исключает корреляцию
- ID агента не передаётся клиенту USER (заменяется на "agent")
- В REST API для пользователя поле `assignee` возвращает `{ displayName: "Ласковый Котёнок" }` вместо реальных данных
