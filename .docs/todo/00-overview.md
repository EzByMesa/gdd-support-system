# GDD Support System — Общий обзор

## Описание проекта
Система тикетов поддержки с веб-интерфейсом, чатом внутри тикетов, группировкой обращений по тематике, анонимными агентами ("милые животные"), делегированием задач, push-уведомлениями, гибкой авторизацией и мастером первоначальной настройки.

## Стек технологий

### Backend
- **Язык:** Node.js (JavaScript, ES Modules)
- **Фреймворк:** Express.js
- **ORM:** Sequelize
- **БД:** PostgreSQL
- **Файлы:** Шифрование AES-256-GCM, хранение без расширений
- **WebSocket:** ws (чат + real-time уведомления)
- **Push-уведомления:** Web Push API (VAPID) + SSE fallback
- **ML-группировка:** @xenova/transformers (all-MiniLM-L6-v2, 384-мерные embeddings)
- **Аутентификация:** JWT (локальный) + refresh tokens

### Frontend
- **Язык:** JavaScript (ES Modules)
- **Сборка:** Vite
- **Фреймворк:** Vanilla JS (без сторонних UI-библиотек)
- **Стили:** CSS (кастомная дизайн-система, пастельные тона, минимализм)
- **Роутинг:** кастомный SPA-роутер
- **Адаптивность:** Mobile-first, полная поддержка десктопа и мобильных (включая админку)

## Роли пользователей

| Роль | Описание | Права |
|------|----------|-------|
| **Корневой Администратор** | Создаётся при Setup Wizard. Может назначать других Администраторов. | Полный доступ. Управление пользователями, ролями, настройками, авторизацией. |
| **Администратор** | Назначается Корневым Админом. Равные права. | То же, что Корневой. Может назначать других Администраторов. |
| **Агент поддержки** | Работает с обращениями. Видит ВСЕ тикеты. | Видит все тикеты (чужие в работе — read-only). Берёт в работу свободные. Делегирует задачи. |
| **Пользователь** | По умолчанию при регистрации. | Создание тикетов. Просмотр своих тикетов. Чат с анонимным агентом. |

## Ключевые механики

### Анонимность агентов
Пользователь НЕ видит реального имени агента. Вместо этого агенту присваивается уникальный псевдоним вида "Прил. + Сущ." из набора милых животных. Подробнее: `11-anonymous-agents.md`

### Делегирование задач
Агент может передать тикет другому агенту. Требуется подтверждение от принимающего. При подтверждении — пользователь получает уведомление о смене "анонимного агента". Подробнее: `12-delegation.md`

### Push-уведомления
Сервер уведомлений на базе Web Push API. Клиент подписывается, получает пуши о: новых сообщениях, смене статуса, делегировании. Подробнее: `13-notifications.md`

### Единый веб-адрес
Один URL для всех ролей. Роутинг определяет интерфейс по роли:
- `/` — для пользователей (список обращений)
- `/admin/...` — отдельный крупный роут с собственным layout для админки

## Этапы реализации

1. **Фаза 0** — Инициализация проекта и инфраструктура
2. **Фаза 1** — Мастер настройки бэкенда (Setup Wizard)
3. **Фаза 2** — Аутентификация и авторизация
4. **Фаза 3** — Тикеты, вложения, видимость для агентов
5. **Фаза 4** — Чат внутри тикетов (WebSocket) + анонимные агенты
6. **Фаза 5** — Делегирование задач
7. **Фаза 6** — Группировка тикетов (Deep Learning)
8. **Фаза 7** — Push-уведомления
9. **Фаза 8** — Админ-панель (отдельный крупный роут)
10. **Фаза 9** — Финализация, адаптив, деплой
