# Фаза 6 — Группировка тикетов по тематике (Deep Learning)

## Концепция

При создании тикета система анализирует его title + description, генерирует embedding-вектор и сравнивает с существующими группами тематик. Если найдена похожая группа — тикет автоматически привязывается. Если нет — создаётся новая группа.

## Библиотека

**TensorFlow.js + Universal Sentence Encoder (USE)**
- Модель: `@tensorflow-models/universal-sentence-encoder`
- Генерирует 512-мерный вектор для текста
- Работает полностью на сервере (Node.js), не требует GPU
- Поддерживает русский язык (multilingual версия)

## Алгоритм группировки

```
1. Получаем текст: title + " " + description
2. Генерируем embedding через USE: vector[512]
3. Сравниваем с embeddings всех существующих TopicGroup:
   - cosine_similarity(new_vector, group_vector)
4. Если max_similarity >= THRESHOLD (0.75):
   - Привязываем тикет к этой группе
   - Пересчитываем embedding группы (среднее всех тикетов)
5. Если max_similarity < THRESHOLD:
   - Создаём новую TopicGroup
   - name = автогенерация из ключевых слов title
   - embedding = vector текущего тикета
6. Обновляем ticketCount группы
```

## Сервис группировки

```javascript
// services/topicGrouping.js

async function classifyTicket(title, description) {
  const text = `${title} ${description}`;
  const embedding = await generateEmbedding(text);

  const groups = await TopicGroup.findAll();

  let bestMatch = null;
  let bestScore = 0;

  for (const group of groups) {
    const score = cosineSimilarity(embedding, group.embedding);
    if (score > bestScore) {
      bestScore = score;
      bestMatch = group;
    }
  }

  if (bestScore >= SIMILARITY_THRESHOLD) {
    return { groupId: bestMatch.id, isNew: false, score: bestScore };
  }

  const newGroup = await TopicGroup.create({
    name: extractKeywords(title),
    embedding: Array.from(embedding),
    autoGenerated: true
  });

  return { groupId: newGroup.id, isNew: true, score: 0 };
}
```

## API эндпоинты

```
GET /api/topic-groups                    — список всех групп
GET /api/topic-groups/:id                — детали группы + тикеты
PUT /api/topic-groups/:id                — редактирование (имя, описание)
POST /api/topic-groups/:id/merge/:otherId — объединить группы
DELETE /api/topic-groups/:id             — удалить группу (тикеты открепляются)
POST /api/topic-groups/reclassify        — перегруппировать все тикеты (admin)
```

## Оптимизация

- Embeddings хранятся как `Float[]` в PostgreSQL
- При большом количестве групп (>1000) — использовать pgvector расширение для быстрого поиска
- Кэширование модели USE в памяти (загрузка при старте ~5 сек)
- Фоновая обработка: классификация через очередь задач, не блокирует создание тикета
